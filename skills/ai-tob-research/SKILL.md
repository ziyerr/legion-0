# AI ToB Research — 中国 AI ToB 全景调研技能

## Overview

**系统化搜索、并行采集、结构化输出。** 将"调研全中国 AI ToB 公司"这种大范围任务拆解为可并行执行的搜索批次，每批覆盖一个细分领域，最终汇总去重输出标准表格。

## 触发条件

用户请求涉及以下任何一种：
- 调研/整理中国 AI ToB 公司
- AI 企业服务市场分析
- AI 创业公司融资盘点
- 特定行业 AI 方案调研
- 竞品/市场全景图整理

## 执行流程

### Phase 0：准备

1. 读取调研目录下已有数据（如有），避免重复劳动
2. 确认输出路径：`{调研目录}/data/` 存分领域原始数据，`{调研目录}/output/` 存最终汇总

### Phase 1：分批并行搜索

**核心原则：每个领域一个搜索批次，批次间并行。**

每个批次（一个领域）执行以下搜索：

```
搜索 1: "{领域} AI 企业服务 融资 2024 2025 2026"
搜索 2: "{领域} 人工智能 解决方案 头部公司"
搜索 3: "{领域} AI 创业公司 客户案例"
搜索 4: "中国 {领域} AI ToB 盘点 排名"
```

**并行策略：**
- 一次最多派 4 个 Agent 并行搜索不同领域
- 每个 Agent 负责 2-3 个相邻领域（减少 Agent 数量）
- Agent 类型：`general-purpose`，带完整搜索工具

**Agent prompt 模板：**

```
你是 AI ToB 调研员，负责搜索以下领域的中国 AI ToB 公司：
- {领域1}（{编号1}）
- {领域2}（{编号2}）

对每个领域，执行以下搜索并提取信息：
1. WebSearch: "{领域} AI 企业服务 融资 2024 2025 2026"
2. WebSearch: "{领域} 人工智能 解决方案 头部公司"  
3. WebSearch: "{领域} AI 创业公司 客户案例 中国"
4. WebSearch: "中国 {领域} AI ToB 盘点"

从搜索结果中提取每家公司的：
- 公司名（全称）
- 产品名
- 主要功能（一句大白话）
- 解决痛点（一句大白话）
- 融资情况（金额或"千万级"等描述，无则标"未披露"）
- 主要客户（具体公司名）
- 行业分类（垂类/通用ToB）
- 细分领域编号

用 Markdown 表格格式输出结果。每家公司一行。
如果搜索结果中提到的公司没有足够信息，仍然记录公司名，缺失字段标"待补充"。

搜索完毕后，将结果写入文件：{输出路径}
```

### Phase 2：汇总去重

所有 Agent 完成后：
1. 读取所有分领域数据文件
2. 按公司名去重（同一公司可能出现在多个领域）
3. 合并多领域信息（一家公司可能同时服务多个行业）
4. 标记信息完整度（全部字段有值 / 部分缺失）

### Phase 3：输出

输出两份文件：
1. **完整表格** `output/ai_tob_全景表.md` — 所有公司，按领域分组
2. **摘要统计** `output/ai_tob_统计.md` — 各领域公司数量、融资总览、头部公司

## 数据质量规则

1. **公司名必须是真实存在的公司** — 不编造，搜不到就不写
2. **融资信息以公开报道为准** — 标注信息来源年份
3. **客户信息以官网/报道为准** — 不推测
4. **一句话描述要大白话** — 不用"赋能""生态""闭环"等空话
5. **去重规则**：同一公司出现在多个领域时，主领域取融资报道中提及的主营方向

## Gotchas

1. **搜索结果有大量重复** — AI大模型公司（百度、阿里、商汤等）会在每个领域都出现，只在 H12 记录一次
2. **融资信息滞后** — 以最新可查的为准，标注年份
3. **有些公司已倒闭/转型** — 搜到负面新闻要标注
4. **不要包含纯 ToC 公司** — 只做消费者产品的不算（如剪映、妙鸭相机）
5. **大厂的 ToB 产品线也算** — 如阿里云通义、百度智能云等，但标注为"大厂ToB部门"
6. **搜索要覆盖 2023-2026** — 很多公司是 2023 年 ChatGPT 之后涌现的
