# 攻击面清单 — 红队必读

红队员在审查代码前，逐条扫描此清单，确保攻击覆盖无遗漏。

## 1. 输入边界

- [ ] **空值/null/undefined** — 函数接收空参数时是否 panic/crash？
- [ ] **超长字符串** — 10K+ 字符的输入是否导致 OOM 或 UI 溢出？
- [ ] **特殊字符** — `< > & " ' \ / \0 \n \r \t` 是否被正确转义？
- [ ] **Unicode 边界** — emoji、零宽字符、RTL 字符、CJK 混排是否正常？
- [ ] **数值边界** — 0、-1、MAX_INT、NaN、Infinity、负数作为 ID/索引？
- [ ] **空集合** — 空数组 `[]`、空对象 `{}` 是否导致 index out of bounds？
- [ ] **类型混淆** — string 传入 number 参数位、object 传入 string 位？

## 2. 路径与文件操作

- [ ] **路径穿越** — `../../etc/passwd`、`..\\..\\` 是否被过滤？
- [ ] **符号链接** — symlink 指向系统文件时操作是否安全？
- [ ] **文件名注入** — 文件名含 `;rm -rf /` 或 `$(cmd)` 时是否被执行？
- [ ] **并发写入** — 两个任务同时写同一文件是否导致损坏？
- [ ] **磁盘满** — 写入失败时是否正确处理而非 crash？
- [ ] **路径长度** — 255+ 字符路径是否被截断或报错？
- [ ] **中文路径** — 项目名含中文时路径拼接是否正确？（本项目常见）

## 3. 命令注入

- [ ] **Shell 注入** — 用户输入拼接到 `Command::new()` / `subprocess.run()` 时是否转义？
- [ ] **Python f-string 注入** — `f"python {user_input}"` 模式？
- [ ] **SQL 注入** — 虽然用 SQLite，但参数化查询是否完整？
- [ ] **环境变量注入** — 环境变量被拼接到命令中？

## 4. 并发与竞态

- [ ] **TOCTOU** — 检查文件存在后再操作，中间被删除？
- [ ] **数据库竞态** — 两个任务同时更新同一条记录？
- [ ] **任务状态竞态** — 任务被取消的同时被完成？
- [ ] **端口竞争** — 两个服务同时绑定同一端口？
- [ ] **文件锁** — 多进程同时访问 tasks.db / project.json？

## 5. 资源耗尽

- [ ] **内存泄漏** — 循环中不断分配不释放？长列表一次性加载到内存？
- [ ] **连接泄漏** — HTTP client 未关闭？数据库连接未归还池？
- [ ] **进程泄漏** — subprocess 启动后未 wait/kill？
- [ ] **临时文件泄漏** — /tmp 写入后未清理？
- [ ] **无限循环** — 重试逻辑没有上限？递归没有终止条件？

## 6. 错误传播

- [ ] **Rust unwrap/expect** — 除了测试代码外是否存在？（项目铁律禁止）
- [ ] **Python bare except** — `except:` 吞掉所有异常？
- [ ] **错误信息泄露** — 错误消息是否暴露文件路径/API key/内部结构？
- [ ] **级联失败** — 上游服务挂掉时下游是否优雅降级？
- [ ] **静默失败** — 错误被捕获但没有日志也没有通知？

## 7. 安全漏洞

- [ ] **XSS** — 用户输入被直接渲染为 HTML（`dangerouslySetInnerHTML`）？
- [ ] **CSRF** — API 端点缺少 token 验证？
- [ ] **敏感信息硬编码** — API key / token / 密码写在代码里？
- [ ] **日志泄露** — 日志中打印了 API key / session token？
- [ ] **未授权访问** — 接口是否缺少权限校验？
- [ ] **不安全的 HTTP** — 外部请求是否用 HTTPS？证书是否验证？

## 8. 回退兼容（本项目特有）

- [ ] **Tauri invoke 参数** — 前端 camelCase ↔ 后端 snake_case 是否一致？
- [ ] **JSON 字段** — 新增/删除字段是否破坏已有 project.json？
- [ ] **Python API** — FastAPI 路由变更是否影响已有调用方？
- [ ] **数据库 schema** — tasks.db 表结构变更是否有迁移？
- [ ] **配置文件** — .env / system_config.json 新字段是否有默认值？

## 9. 异步与超时

- [ ] **无超时请求** — HTTP 调用没设 timeout，目标服务挂了就永远等？
- [ ] **异步泄漏** — async 任务 spawn 后未 await/abort？
- [ ] **回调地狱** — Promise 链中间某一环 reject 了，后续怎么处理？
- [ ] **取消传播** — 用户取消操作时，底层任务是否真的停止了？

## 使用方法

1. 先读 git diff 了解改了什么
2. 对照此清单，**只检查与改动相关的类别**
3. 每发现一个问题，记录：文件:行号 + 攻击方式 + 影响
4. 清单中打钩表示"已检查，无问题"，不打钩表示"不适用于本次改动"
