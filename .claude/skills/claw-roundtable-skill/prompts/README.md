# RoundTable 提示词框架 v2.0

## 📋 概述

本框架采用**通用流程 + 行业模板**的设计，确保 5 轮 RoundTable 讨论后能产出**初步可执行**的方案（可行性 80%+）。

- **通用框架**：多视角讨论、批判性思维、达成共识的流程（所有行业复用）
- **行业模板**：可执行方案的具体产出物定义（分行业定制）

---

## 🏗️ 框架结构

```
prompts/
├── framework.md              # 通用框架（本文件）

templates/
├── software-development.md   # 软件开发模板
├── product-planning.md       # 产品规划模板
└── business-research.md      # 商业研究模板
```

---

## 🔄 5 轮流程

| 轮次 | 主题 | 核心任务 | 字数要求 |
|------|------|---------|---------|
| R1 | 独立方案 | 提供专业方案 + 量化指标 | 400-600 字 |
| R2 | 相互引用 | 引用 2 位专家 + 指出 3 个缺失 + 补充 2 个建议 | 400-600 字 |
| R3 | 深度批判 | 识别 3 个风险 + 批判方案 + 改进建议 | 400-600 字 |
| R4 | 辩论完善 | 回应质疑 + 辩论观点 + 寻求共识 | 400-600 字 |
| R5 | 虾软总结 | 汇总讨论 + 产出可执行文档 | 400-600 字 |

---

## 📐 通用提示词模板

### R1 轮：独立方案

```markdown
你是一位资深{role}专家。

## 讨论主题
{topic}

## R1 轮任务：独立方案

请用**400-600 字**提供专业方案，有话说话，不要凑字数。

### 必须包含
1. **核心观点**（100-200 字）
   - 你对主题的核心判断
   - 关键依据（数据/案例/经验）

2. **具体方案**（200-300 字）
   - {industry_specific_1}
   - {industry_specific_2}

3. **量化指标**（50-150 字）
   - 至少 3 个可量化的目标
   - 每个指标必须有测量方法

## 输出要求
- **字数：400-600 字**
- 使用 Markdown 格式
- 包含必要的表格
- **不要为了凑字数而重复**

请开始：
```

### R2 轮：相互引用 + 补充

```markdown
你是一位资深{role}专家。

## R2 轮任务：相互引用 + 补充

请阅读其他专家的 R1 方案，然后：

1. **引用回应**（150-250 字）
   - 赞同或质疑其他专家的观点（至少引用 2 位）
   - 说明理由

2. **缺失识别**（150-250 字）
   - 指出当前方案中**至少 3 个缺失的关键信息**
   - 说明为什么这些信息对可执行性重要

3. **补充建议**（100-200 字）
   - 提出**至少 2 个具体的补充方案**

## 输出要求
- **字数：400-600 字**
- 明确标注引用的专家
- 有话说话，不要凑字数

请开始：
```

---

### R3 轮：深度分析 + 批判

```markdown
你是一位资深{role}专家。

## R3 轮任务：深度分析 + 批判

基于 R1+R2 轮讨论，请进行深度批判性分析：

1. **风险识别**（150-250 字）
   - 指出当前方案中的风险（至少 3 个）
   - 每个风险标注影响程度（高/中/低）

2. **方案批判**（150-250 字）
   - 质疑其他人的观点，提出不同意见
   - 说明你的理由和依据

3. **改进建议**（100-200 字）
   - 针对风险提出具体改进方案
   - 每个建议必须有可执行性

## 输出要求
- **字数：400-600 字**
- 有话说话，不要凑字数
- 展示批判性思维

请开始：
```

---

### R4 轮：辩论完善

```markdown
你是一位资深{role}专家。

## R4 轮任务：辩论完善

基于 R3 轮的批判，请：

1. **回应质疑**（150-250 字）
   - 回应其他人对你方案的质疑
   - 说明你的立场和理由

2. **辩论观点**（150-250 字）
   - 对仍有分歧的点进行辩论
   - 提出你的折中方案

3. **寻求共识**（100-200 字）
   - 提出可落地的折中方案
   - 说明各方需要做出的妥协

## 输出要求
- **字数：400-600 字**
- 有话说话，不要凑字数
- 展示辩论和妥协

请开始：
```

## 📁 行业模板

### 1. 软件开发（software-development）

**适用场景**: 系统设计、技术选型、架构评审、功能开发

**专家角色**:
- 🔧 工程专家
- 🎨 体验专家
- 🧪 质量专家
- 📊 产品专家

**产出文档**:
- 技术规格说明书（ER 图 + API 清单 + 模块依赖）
- 体验设计规范（用户流程 + Design Token + 线框图）
- 测试计划文档（测试场景 + 性能基准 + 混沌工程）
- 产品需求文档（MVP 清单 + 验收标准 + 埋点）
- 行动计划（责任人 + 时间线 + 交付物）

---

### 2. 产品规划（product-planning）

**适用场景**: 新产品立项、功能规划、产品迭代、商业模式设计

**专家角色**:
- 📈 市场专家
- 📊 产品专家
- 💰 商业专家
- 🚀 运营专家

**产出文档**:
- 产品定位文档（用户画像 + 价值主张 + 差异化）
- 功能规划文档（优先级矩阵 + MVP 清单 + 路线图）
- 商业模型文档（收入模式 + 定价 + 成本 + 盈亏平衡）
- 市场分析报告（TAM/SAM/SOM + 竞争格局）
- 运营计划文档（冷启动 + 获客 + 转化 + 留存）
- 风险评估文档（市场/竞争/执行风险 + 应对预案）
- 行动计划（0-30 天 + 30-90 天 + 90-180 天）

---

### 3. 商业研究（business-research）

**适用场景**: 市场调研、竞争分析、投资机会评估、行业研究

**专家角色**:
- 🏭 行业专家
- 📊 市场专家
- ⚔️ 竞争专家
- 💵 投资专家

**产出文档**:
- 行业分析报告（产业链 + 市场规模 + 趋势 + 政策）
- 市场分析报告（细分 + 用户画像 + 需求痛点）
- 竞争格局分析（玩家清单 + 份额 + 对比矩阵）
- 投资机会评估（机会清单 + ROI + 风险 + 退出路径）
- 标杆案例研究（成功案例 + 失败案例）
- 战略建议报告（进入策略 + 短中长期规划）
- 数据附录（数据来源 + 调研方法 + 样本说明）

## 🎯 可执行性验证

每个模板都包含**可执行性检查清单**，在 R5 轮完成后自动验证：

### 软件开发检查项
- [ ] 数据库 ER 图是否包含核心表（至少 5 张表）
- [ ] API 接口清单是否包含核心端点（至少 10 个）
- [ ] 测试场景是否覆盖核心流程（至少 10 个用例）
- [ ] MVP 功能是否有验收标准（每个功能）
- [ ] 时间线是否具体到周（周粒度）
- [ ] 责任人是否明确（每个任务）

### 产品规划检查项
- [ ] 目标用户画像是否具体（有姓名/职业/痛点）
- [ ] 功能优先级是否有依据（重要/紧急矩阵）
- [ ] 定价是否有竞品对比依据
- [ ] 盈亏平衡点是否有计算过程
- [ ] 冷启动策略是否具体到渠道和预算
- [ ] 行动计划是否具体到周（0-30 天）

### 商业研究检查项
- [ ] 市场规模是否有数据来源（可追溯）
- [ ] 竞品对比是否有具体维度（非笼统描述）
- [ ] 用户画像是否具体（有姓名/职业/行为）
- [ ] 投资机会是否有量化评估（金额/ROI）
- [ ] 行动计划是否具体到月（0-6 个月）
- [ ] 数据来源是否完整（所有引用有出处）

**可行性目标**: 80% 以上（相关团队可直接执行）

## 🚀 使用说明

### 发起 RoundTable 讨论

```python
# 示例：软件开发 RoundTable
roundtable.run(
    topic="在线点餐系统 - 支持 Web/小程序/App 多端",
    industry="software-development",
    rounds=5,
    word_limit="400-600"
)

# 示例：产品规划 RoundTable
roundtable.run(
    topic="AI 笔记产品立项",
    industry="product-planning",
    rounds=5,
    word_limit="400-600"
)

# 示例：商业研究 RoundTable
roundtable.run(
    topic="中国预制菜市场研究",
    industry="business-research",
    rounds=5,
    word_limit="400-600"
)
```

### 添加新行业模板

1. 在 `templates/` 目录创建新模板文件
2. 定义行业信息、专家角色、产出文档
3. 定义 R1 轮行业特定要求
4. 定义可执行性检查清单
