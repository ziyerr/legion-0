# RoundTable 提示词框架 v3.0

## 🎯 核心改进（基于 2026-03-19 真实测试）

### 本次测试验证的关键成功要素
1. ✅ **真实子 Agent 调用** - 15 次独立 sessions_spawn，每轮内容不同
2. ✅ **批判性思维激活** - 15 个风险 + 9 大缺陷，敢于质疑
3. ✅ **方案动态演进** - 工时 +71%、策略重大调整，吸收批评
4. ✅ **分歧明确裁决** - 4 项分歧 + 理由，不回避矛盾
5. ✅ **产出可直接执行** - W1-W8 任务表 + 风险预案

### v3.0 vs v2.0 关键差异
| 改进点 | v2.0 问题 | v3.0 解决方案 |
|--------|----------|--------------|
| **上下文传递** | R2-R5 无前几轮内容 | **强制注入完整讨论历史** |
| **批判深度** | 泛泛而谈（3 个风险） | **强制 5 个风险 + 3 个缺陷** |
| **方案演进** | R1-R4 内容重复 | **R4 必须标注修改内容 + 理由** |
| **分歧处理** | 无裁决机制 | **R5 必须裁决仍存分歧** |
| **输出质量** | 400-600 字、无表格 | **800-2000 字 + 强制表格** |

## 🔄 5 轮流程（强化版）

| 轮次 | 主题 | 核心任务 | 字数要求 | 强制要求 |
|------|------|---------|---------|---------|
| R1 | 独立方案 | 专业方案 + 量化指标 | **800-1200 字** | 至少 1 个对比表格 |
| R2 | 相互引用 | **明确引用 2 位专家** + 补充 | **800-1200 字** | 至少 3 处引用标注 |
| R3 | 深度批判 | **5 个风险 + 3 大缺陷** | **1000-1500 字** | 风险表格 + 数据支撑 |
| R4 | 辩论完善 | **回应所有质疑 + 标注修改** | **1000-1500 字** | 修改对比表 |
| R5 | 虾软总结 | **裁决分歧 + 周级计划** | **1500-2000 字** | 8 周计划表 + 风险预案 |

## 🚀 使用说明

### 发起 RoundTable 讨论（真实子 Agent 模式）

```python
# 关键：每轮必须传递完整讨论历史
async def run_roundtable(topic: str, industry: str):
    history = {}
    
    # R1: 独立方案
    history['R1'] = await spawn_subagent(
        role='工程专家',
        prompt=build_r1_prompt(topic, industry),
        label='rt-r1-engineering'
    )
    
    # R2: 相互引用（必须传入 R1 历史）
    history['R2'] = await spawn_subagent(
        role='工程专家',
        prompt=build_r2_prompt(topic, history['R1']),
        label='rt-r2-engineering'
    )
    
    # R3-R5: 同上，每轮传入完整历史
    
    return history
```

## 🎯 目标

通过 5 轮 RoundTable 讨论，产出**可行性 90%+** 的可执行方案。

**核心原则**：
1. **真实子 Agent 调用** - 每轮独立思考，内容不重复
2. **强制批判深度** - 5 个风险 + 3 个缺陷
3. **方案动态演进** - R4 必须标注修改
4. **分歧明确裁决** - R5 必须裁决分歧
5. **产出可直接执行** - 周级计划 + 风险预案
