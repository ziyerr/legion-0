# 产品规划行业模板

## 行业信息
- **industry**: product-planning
- **名称**: 产品规划
- **适用场景**: 新产品立项、功能规划、产品迭代、商业模式设计

---

## 专家角色（4 位）

1. **市场专家**（📈）
   - 职责：市场分析、竞争格局、机会识别
   - R1 重点：市场规模、增长趋势、细分市场

2. **产品专家**（📊）
   - 职责：产品定位、功能规划、用户体验
   - R1 重点：目标用户、核心价值、差异化

3. **商业专家**（💰）
   - 职责：商业模式、收入预测、成本结构
   - R1 重点：收入模式、成本分析、盈亏平衡

4. **运营专家**（🚀）
   - 职责：冷启动策略、增长计划、运营体系
   - R1 重点：获客渠道、转化漏斗、留存策略

---

## 必须产出文档（R5 轮）

### 1. 产品定位文档
**负责人**: 产品专家
**必须包含**:
- 目标用户画像（至少 3 个典型用户角色）
- 核心价值主张（一句话说清为什么选你）
- 差异化分析（与竞品的 3-5 个关键差异）
- 使用场景描述（至少 5 个典型使用场景）

### 2. 功能规划文档
**负责人**: 产品专家
**必须包含**:
- 功能优先级矩阵（重要/紧急四象限）
- MVP 功能清单（P0/P1/P2 优先级排序）
- 功能依赖关系图（哪些功能必须先做）
- 迭代路线图（V1.0→V1.5→V2.0 时间线）

### 3. 商业模型文档
**负责人**: 商业专家
**必须包含**:
- 收入模式（至少 3 种收入来源）
- 定价策略（具体价格档位和依据）
- 成本结构（固定成本 + 变动成本明细）
- 盈亏平衡分析（多少用户/收入可盈亏平衡）
- 财务预测（12 个月收入和利润预测）

### 4. 市场分析报告
**负责人**: 市场专家
**必须包含**:
- 市场规模（TAM/SAM/SOM 三层分析）
- 增长趋势（年复合增长率 CAGR）
- 主要玩家分析（至少 5 个竞品对比）
- 市场机会清单（至少 3 个可切入的机会点）
- 进入壁垒分析（技术/资金/渠道/品牌）

### 5. 运营计划文档
**负责人**: 运营专家
**必须包含**:
- 冷启动策略（首批 100/1000/10000 用户获取方案）
- 获客渠道分析（至少 5 个渠道的 ROI 预估）
- 转化漏斗设计（从知晓到付费的转化率目标）
- 留存策略（次日/7 日/30 日留存目标）
- 关键运营指标（DAU/MAU/LTV/CAC 目标值）

### 6. 风险评估文档
**负责人**: 虾软 Host
**必须包含**:
- 市场风险（需求不成立/市场太小/增长太慢）
- 竞争风险（巨头进入/价格战/被复制）
- 执行风险（团队能力/资金不足/延期）
- 每个风险的应对预案（至少 2 个应对措施）

### 7. 行动计划
**负责人**: 虾软 Host
**必须包含**:
- 0-30 天行动清单（具体到每周）
- 30-90 天里程碑（可量化的阶段目标）
- 90-180 天规划（中长期目标）
- 责任人分配（每个任务的负责人）
- 资源需求（人力/资金/外部支持）

---

## R1 轮行业特定要求

### 市场专家
- 市场规模数据（TAM/SAM/SOM）
- 竞争格局分析（市场份额 + 主要玩家）
- 市场机会识别（至少 3 个切入点）

### 产品专家
- 目标用户画像（人口统计 + 行为特征）
- 核心价值主张（一句话说清差异化）
- MVP 功能清单（优先级排序）

### 商业专家
- 收入模式设计（至少 3 种来源）
- 定价策略（具体价格档位）
- 成本结构分析（固定 + 变动成本）

### 运营专家
- 冷启动策略（首批用户获取方案）
- 获客渠道分析（至少 5 个渠道）
- 转化漏斗设计（各环节转化率目标）

---

## 可执行性检查清单

在 R5 轮完成后，自动验证：
- [ ] 目标用户画像是否具体（有姓名/职业/痛点）
- [ ] 功能优先级是否有依据（重要/紧急矩阵）
- [ ] 定价是否有竞品对比依据
- [ ] 盈亏平衡点是否有计算过程
- [ ] 冷启动策略是否具体到渠道和预算
- [ ] 行动计划是否具体到周（0-30 天）
- [ ] 责任人是否明确（每个任务）

**可行性目标**: 80% 以上（产品团队可直接执行）
