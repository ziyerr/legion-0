# 软件开发行业模板

## 行业信息
- **industry**: software-development
- **名称**: 软件开发
- **适用场景**: 系统设计、技术选型、架构评审、功能开发

---

## 专家角色（4 位）

1. **工程专家**（🔧）
   - 职责：技术架构、技术选型、工时评估
   - R1 重点：技术栈对比、模块划分、交付周期

2. **体验专家**（🎨）
   - 职责：用户体验、界面设计、交互流程
   - R1 重点：用户画像、核心流程、视觉规范

3. **质量专家**（🧪）
   - 职责：测试策略、质量保障、性能指标
   - R1 重点：测试金字塔、性能基准、质量门禁

4. **产品专家**（📊）
   - 职责：产品定位、功能规划、商业模式
   - R1 重点：市场分析、MVP 定义、迭代节奏

---

## 必须产出文档（R5 轮）

### 1. 技术规格说明书
**负责人**: 工程专家
**必须包含**:
- 数据库 ER 图（核心表结构）
- API 接口清单（核心端点定义：方法/路径/参数/返回值）
- 模块依赖关系图（模块列表 + 接口定义）
- 第三方服务选型（支付/短信/地图等服务商和接入方案）

### 2. 体验设计规范
**负责人**: 体验专家
**必须包含**:
- 核心用户流程图（至少 3 个关键流程）
- Design Token 定义（色彩/字体/间距规范）
- 关键界面线框图描述（至少 5 个核心页面）
- 多端适配策略（Web/小程序/App 设计要点）

### 3. 测试计划文档
**负责人**: 质量专家
**必须包含**:
- 核心测试场景清单（至少 10 个测试用例）
- 性能基准数据（P99/TPS 目标值 + 压测场景）
- 混沌工程场景（至少 3 个故障注入场景）
- 测试左移准出指标（覆盖率/静态扫描/自动化通过标准）

### 4. 产品需求文档
**负责人**: 产品专家
**必须包含**:
- MVP 功能清单（优先级排序：P0/P1/P2）
- 每个功能的验收标准（可量化的完成定义）
- 数据埋点清单（核心事件 + 转化漏斗）
- 迭代路线图（V1.0→V1.5→V2.0 功能规划）

### 5. 行动计划
**负责人**: 虾软 Host
**必须包含**:
- 责任人分配（每个任务的负责人）
- 时间线（周粒度，含里程碑）
- 交付物清单（每个阶段的输出）
- 风险应对预案（Top 5 风险 + 应对措施）

---

## R1 轮行业特定要求

### 工程专家
- 技术选型对比表（至少 3 个选项的优劣对比）
- 模块边界和接口定义（输入/输出）
- 里程碑和交付物定义

### 体验专家
- 用户画像和核心痛点（至少 3 个）
- 核心流程设计（用户/商家/运营三端）
- 视觉风格定义（主色/辅助色/字体）

### 质量专家
- 测试策略和自动化比例
- 关键测试场景（下单/支付/并发）
- 性能指标（P99/TPS/错误率目标值）

### 产品专家
- 市场规模和竞争格局
- 产品定位和差异化优势
- MVP 功能清单和优先级表格

---

## 可执行性检查清单

在 R5 轮完成后，自动验证：
- [ ] 数据库 ER 图是否包含核心表（至少 5 张表）
- [ ] API 接口清单是否包含核心端点（至少 10 个）
- [ ] 测试场景是否覆盖核心流程（至少 10 个用例）
- [ ] MVP 功能是否有验收标准（每个功能）
- [ ] 时间线是否具体到周（周粒度）
- [ ] 责任人是否明确（每个任务）

**可行性目标**: 80% 以上（开发团队可直接执行）
