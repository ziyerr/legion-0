# 审查者速查清单

审查每个文件时逐条过，不漏项。打钩 = 检查过无问题，标注 = 有问题需反馈。

## 一、正确性（最高优先级）

- [ ] **需求符合** — 代码是否实现了用户要的功能？不多不少？
- [ ] **逻辑正确** — 条件判断、循环边界、状态转换是否正确？
- [ ] **数据流** — 输入→处理→输出，每一步数据变换是否正确？
- [ ] **边界处理** — 空值、零值、空数组、超大输入是否处理？
- [ ] **错误路径** — 错误发生时执行流是否合理？是否有未处理的异常？

## 二、安全性（不可妥协）

- [ ] **无 unwrap** — Rust 代码中除测试外禁止 unwrap（项目铁律）
- [ ] **无注入** — 用户输入不直接拼接到命令/SQL/HTML 中
- [ ] **无硬编码密钥** — API key/token/密码不在代码中
- [ ] **无 XSS** — 用户内容不通过 dangerouslySetInnerHTML 渲染
- [ ] **路径安全** — 文件路径操作防止穿越（../../）

## 三、项目规范

- [ ] **Tauri 参数约定** — 前端 invoke camelCase，后端 snake_case + serde(rename_all = "camelCase")
- [ ] **错误处理** — Rust 用 `?` 或 `map_err`，不用 unwrap
- [ ] **文件隔离** — 只修改了分配范围内的文件
- [ ] **import 整洁** — 无未使用的 import，无循环依赖

## 四、设计质量

- [ ] **单一职责** — 函数/组件是否只做一件事？
- [ ] **命名清晰** — 变量/函数名是否自解释？
- [ ] **无重复** — 是否有可提取的公共逻辑？（但三行以内的重复不必提取）
- [ ] **接口一致** — 新增 API 的命名和参数风格是否与已有接口一致？

## 五、前端专项（仅 .tsx/.css 文件）

- [ ] **状态管理** — state 是否在正确的层级？是否有不必要的重渲染？
- [ ] **key 属性** — 列表渲染是否有稳定唯一的 key？
- [ ] **副作用清理** — useEffect 的清理函数是否正确？
- [ ] **类型安全** — TypeScript 类型是否准确？是否有 any 逃逸？
- [ ] **CSS 隔离** — 样式是否有意外的全局影响？

## 六、Rust 专项（仅 .rs 文件）

- [ ] **所有权** — clone 是否必要？能否用引用？
- [ ] **异步安全** — async 函数中是否持有锁跨越 await？
- [ ] **错误类型** — 自定义错误是否实现了必要的 trait？
- [ ] **序列化** — Tauri command 的参数和返回值 serde 注解是否正确？

## 七、Python 专项（仅 .py 文件）

- [ ] **异步一致** — async 函数是否被正确 await？同步调用是否阻塞事件循环？
- [ ] **资源释放** — 文件/HTTP 连接是否用 with/async with？
- [ ] **类型提示** — 关键函数是否有类型注解？
- [ ] **日志** — 关键操作是否有日志？日志是否泄露敏感信息？

## 审查反馈格式

发现问题时：
```
文件:行号 — [严重/建议] 问题描述 + 修改建议
```

例：
```
gui/src/App.tsx:42 — [严重] invoke 参数用了 snake_case，应改为 camelCase
gui/src-tauri/src/commands/project.rs:128 — [严重] unwrap 违反项目铁律，改用 .map_err()?
scripts/video_gen.py:56 — [建议] HTTP 请求没设 timeout，建议加 timeout=30
```

全部通过时：
```
文件名 APPROVED
```
