# Sniper — 狙击手（定点清除）

## Overview

**修一次是修 bug，修两次是灭火，修三次是失职。**

当同一问题或同类问题反复出现时，常规修复已经失效。狙击手的任务是跳出"修了又来"的循环，找到结构性根因，定点清除。

```
常规修复循环（治标）：
  问题出现 → 修复 → 关闭 → 复发 → 修复 → 关闭 → 复发 ...

狙击手介入（治本）：
  第 2 次复发 → 暂停修复 → 根因分析 → 定点清除 → 永久消除
```

## 触发条件

指挥官判断以下**任一**条件满足时，召唤狙击手：

1. **同一问题复发** — 相同文件/模块的相同类型错误出现 ≥ 2 次
2. **同类问题丛生** — 不同位置但模式相同的问题出现 ≥ 3 处
3. **修了又坏** — 某个修复被后续改动破坏（回归）
4. **战法库有记录但还是踩坑** — 说明战法没被执行到位，流程有漏洞
5. **用户指出** — "这个问题怎么又来了"

## 不触发的情况

- 首次出现的新问题 → 正常修复流程
- 不同根因导致的表面相似问题 → 正常修复
- 外部依赖变更导致的问题 → 技术参谋侦察

## 召唤方式

狙击手通过 `subagent_type: "sniper"` 创建：

```
Agent(
  subagent_type: "sniper",
  name: "sniper",
  model: "opus",
  prompt: """
    首先执行：/effort max

    你是狙击手（定点清除）。

    【当前问题】
    {描述当前问题}

    【历史线索】
    - 第 1 次出现：{时间/上下文/当时的修复}
    - 第 2 次出现：{时间/上下文/当时的修复}
    - 当前（第 N 次）：{触发场景}

    【涉及区域】
    {文件路径/模块/功能域}

    读 .Codex/agents/sniper.md 了解你的职责和分析方法。
    完成后将根因分析报告通过 SendMessage 发给指挥官。
  """
)
```

## 空降伞兵联合狙杀（可选，L/XL 级推荐）

差异模型对同一问题的根因判断可能不同。伞兵（Codex/GPT）的不同认知框架可能发现 Codex 忽略的结构性问题。

```bash
# 狙击手分析后，空降伞兵做独立交叉验证
bash .Codex/scripts/codex-team.sh second-opinion \
  "以下问题反复出现 N 次：[描述]。
   Codex 团队认为根因是：[根因]。
   请独立分析，你认为根因是什么？是否同意 Codex 的判断？"
```

**处理规则：**
- 狙击手和伞兵根因判断一致 → 高置信度，直接执行
- 不一致 → 指挥官综合判断，可能两个都对（多根因），也可能需要更深调查

## 指挥官编排流程

```
1. 发现问题复发（≥2 次）
2. 暂停！不要急着修。
3. 收集历史线索（git log, 战法库, 团队记忆）
4. 创建狙击手 teammate
5. (可选) 同时调 Codex second-opinion 交叉验证
6. 收到根因分析报告
7. 审阅：根因是否通过反事实验证？
   ├── 通过 → 交给实现者做结构性修复
   └── 未通过 → 反馈给狙击手补充调查
8. 结构性修复完成后：
   - 更新/新建战法（~/.Codex/memory/tactics/）
   - 如有必要，更新审查清单或 hook
```

## 与现有角色的协作

| 场景 | 谁做什么 |
|------|---------|
| 首次 bug | 实现者修 + 审查者查 → 正常流程 |
| 复发 bug（≥2次） | **狙击手分析** → 实现者做结构性修复 → 审查者查 |
| 复发 + 关键系统 | 狙击手 + **伞兵交叉验证** → 实现者修 → 验证者全量验证 |

## 产出去向

狙击手的报告最终变成三样东西：

1. **结构性修复** — 实现者根据方案改代码（代码层防御优先）
2. **战法沉淀** — 新增到 `~/.Codex/memory/tactics/`（未来不再踩坑）
3. **审查清单更新** — 如果发现审查环节有盲区，更新 verify agent 的检查项

## Gotchas

1. **不要急着修** — 复发问题最忌讳的是"先修了再说"，那只是第 N+1 次灭火
2. **历史证据必须充分** — git log/blame 是最可靠的证据，记忆可能有偏差
3. **根因可能不在代码里** — 可能是流程问题（审查遗漏）、工具问题（hook 没覆盖）、认知问题（团队不知道某个约束）
4. **反事实检验是硬标准** — "感觉是这个原因"不够，必须过反事实
5. **防复发方案要分层** — 代码层 > 工具层 > 流程层，越底层越可靠
6. **战法不是写了就完了** — 战法写了但没人看 = 没写。考虑是否需要加 hook 自动检查
