summary: 质量门控必须区分全损(0%)与部分不达标(>0%<阈值)，全损走 bug 升级而非重试
---

当质量评分为 0%（或覆盖率为 0%）时，根因几乎一定是脚本异常、输入格式不匹配或依赖缺失，而非 LLM 输出质量波动。此时用相同参数重试只会浪费重试配额。正确做法：(1) 全损（=0%）→ 标记为 BUG，捕获 stderr/traceback，升级给开发者或指挥官；(2) 部分不达标（>0% 但 <阈值）→ 附带改进指导正常重试。判断阈值建议：质量 < 10% 视为全损。
