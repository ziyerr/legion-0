summary: 媒体处理管道的端到端测试必须覆盖到最终用户可访问的URL，不能止步于处理完成
---

视频/音频/图片处理管道通常有隐式的发布/分发步骤（如状态切换、CDN刷新、签名URL生成）。测试脚本容易只验证到'处理成功'就结束，遗漏发布和可访问性验证。正确做法：测试最后一步必须用 HTTP HEAD 验证最终URL返回200且content-length合理，而非信任API返回的'成功'状态。
